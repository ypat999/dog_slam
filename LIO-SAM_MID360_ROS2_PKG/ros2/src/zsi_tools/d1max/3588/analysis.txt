3588 板子是智元机器狗的**主控制器**，运行瑞芯微 RK3588 芯片，负责
机器人的底层运动控制、硬件抽象、传感器接入等功能，与 zg-arc 板子
（运行上层算法）协同工作。

======================================================================
  bin/ 目录脚本分析
======================================================================

一、系统启动与初始化
----------------------------------------------------------------------

1. robot_start.sh
   - 用途: 3588 板子上电初始化脚本
   - 功能:
     * 迁移 device.json 到新路径
     * 清理残留的升级/OTA 目录
     * 调优 SoC 温度阈值 (100°C/95°C/120°C)
     * 设置 CPUFreq 为 performance 模式 (降低延迟)
     * 等待时间同步完成后继续启动

2. robot_start_app.sh
   - 用途: 机器人应用层服务启动
   - 功能: 创建日志目录（保留最近 5 份），启动 Zenoh 路由器，
           启动 robot-launch server 进程

3. robot_wait_time_sync.sh
   - 用途: 等待时间同步（与 zg-arc 相同）
   - 功能: 通过 chronyc 检查 Leap status，最多等待 25 秒

4. start_zenoh_router.sh
   - 用途: Zenoh 通信中间件路由器


二、运动控制与底层驱动
----------------------------------------------------------------------

5. start_motion_control.sh
   - 用途: 运动控制器 (MC - Motion Control)
   - 功能:
     * 等待 /dev/shm/joint_cmd 设备就绪
     * 配置 lo 网卡组播路由
     * 解析设备类型 (ZSM-1 / ZSM-1F)
     * 执行 mc_ctrl 运动控制程序，绑定 CPU core 7
   - 设备类型:
     - ZSM-1:ZGWS (四足机器狗)
     - ZSM-1F:ZG (改进型)
   - 共享内存设备: /dev/shm/joint_cmd, /dev/shm/joint_state

6. start_robot_hal.sh
   - 用途: 硬件抽象层 (HAL - Hardware Abstraction Layer)
   - 功能: 启动 robot_hal 节点，根据设备类型加载不同配置
     * ZSM-1 -> zs_m1.launch.py
     * ZSM-1F -> zs_m1f.launch.py
   - 负责: 将底层硬件抽象为统一的 ROS2 接口


三、相机与视频服务
----------------------------------------------------------------------

7. start_robot_camera.sh
   - 用途: 相机服务启动
   - 功能:
     * 等待 /dev/video22 和 /dev/video31 设备就绪
     * 配置 v4l2 视频格式 (1920x1080, NV12)
     * 配置垂直/水平翻转
     * 启动 mediamtx RTSP 服务器 (mediamtx.yml)
     * 启动 robot_camera 节点 (zsm.launch.py)
   - 视频流:
     * 前置相机: /dev/video22
     * 后置相机: /dev/video31

8. start_gst_video_front.sh / start_gst_video_back.sh
   - 用途: GStreamer 视频流启动脚本 (前后相机)


四、遥控与通信
----------------------------------------------------------------------

9. start_robot_remote.sh
   - 用途: 遥控接收器服务
   - 功能: 启动 robot_remote 节点，根据设备类型加载配置
     * ZSM-1 -> zs_m1.launch.py
     * ZSM-1F -> zs_m1f.launch.py
   - 负责: 处理遥控器信号输入


五、监控与诊断

======================================================================
  系统架构总结
======================================================================

3588 板子采用"底层控制 + 抽象层"架构：

  底层硬件
    │
    ├─ 运动控制 ──> mc_ctrl (CPU core 7)
    ├─ 执行器 ────> zsi_actuator_driver
    ├─ 电池 ──────> battery_controller
    ├─ LED ───────> led_controller
    ├─ IMU ───────> zsi_imu_driver / imu_publisher
    ├─ 相机 ──────> robot_camera + mediamtx
    └─ 遥控 ──────> robot_remote / sbus_controller

  硬件抽象层 (HAL)
    │
    └─ robot_hal (zs_m1.launch.py / zs_m1f.launch.py)

  应用层
    │
    ├─ robot_manager    - 整体管理
    ├─ robot_monitor    - 状态监控
    ├─ robot_diagnostic_analyzer - 诊断分析

  通信层
    │
    ├─ Zenoh 中间件
    └─ 与 zg-arc 板子通信

  网络服务
    │
    ├─ WiFi STA/AP 模式
    ├─ RTSP 视频流 (mediamtx)
    └─ 域桥接

======================================================================
  启动流程
======================================================================

1. robot_start.sh (3588 板子初始化)
   - 清理环境
   - 调优 CPU/热管理
   - 等待时间同步

2. robot_start_app.sh (启动应用服务)
   - 启动 Zenoh 路由器
   - 启动 robot-launch server

3. 启动各个服务:
   - start_motion_control.sh   - 运动控制
   - start_robot_hal.sh        - 硬件抽象层
   - start_robot_camera.sh     - 相机 + RTSP
   - start_robot_remote.sh     - 遥控
   - start_robot_manager.sh    - 管理器
   - start_robot_monitor.sh    - 监控
   - start_robot_diagnostic_analyzer.sh - 诊断
